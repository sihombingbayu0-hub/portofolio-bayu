01 / Tentang Project

Otomatisasi sederhana untuk penggunaan yang lebih higienis.

Project ini dibuat untuk mengurangi kontak langsung dengan penutup tempat sampah sekaligus memberikan informasi ketika kapasitas sudah penuh.

Latar belakang dan tujuan

Tempat sampah manual mengharuskan pengguna menyentuh penutup dan sering tidak memberi tanda saat isinya sudah penuh. Prototype ini menggabungkan dua sensor jarak, aktuator servo, audio, serta Telegram agar proses membuang sampah lebih praktis dan kondisi kapasitas lebih mudah diketahui.

NodeMCU membaca sensor pengguna dan sensor kapasitas secara berkala. Tutup hanya dibuka jika pengguna berada di bawah 50 cm dan status tempat sampah belum penuh.

Manfaat

  • Mengurangi sentuhan langsung pada penutup.
  • Memberikan peringatan audio sesuai kondisi.
  • Mengirim status penuh dan kosong melalui Telegram.
  • Menahan proses pembukaan ketika kapasitas penuh.

Peran saya

  • Merancang rangkaian perangkat keras.
  • Memprogram NodeMCU ESP8266.
  • Mengintegrasikan sensor, servo, DFPlayer, dan Telegram.
  • Menguji sensor serta alur sistem.
  • Membuat prototype dan dokumentasi project.

03 / Cara Kerja Sistem

Dari deteksi pengguna sampai notifikasi penuh.

  1. 01

    Mendeteksi pengguna

    Sensor ultrasonik pertama membaca jarak orang di depan penutup.

  2. 02

    Memeriksa kapasitas

    Sistem memastikan status tempat sampah belum penuh sebelum membuka.

  3. 03

    Membuka penutup

    Servo bergerak perlahan dari 165° menuju 75°.

  4. 04

    Memutar audio

    DFPlayer Mini menjalankan file /mp3/0001.mp3.

  5. 05

    Menutup kembali

    Setelah pengguna menjauh selama 2 detik, servo kembali ke posisi tertutup.

  6. 06

    Memantau isi

    Sensor kedua membaca kapasitas setiap siklus sensor sekitar 300 ms.

  7. 07

    Mengunci saat penuh

    Tiga pembacaan di bawah 4 cm mengaktifkan status penuh dan mencegah pembukaan.

  8. 08

    Mengirim Telegram

    Sistem mengirim peringatan penuh dan status siap setelah jarak reset melewati 15 cm.

02Pembacaan sensor ultrasonikMenghasilkan jarak valid 2–400 cm 
float bacaJarakCm(uint8_t trigPin, uint8_t echoPin) {
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);

  unsigned long durasi = pulseIn(echoPin, HIGH, 30000UL);
  if (durasi == 0) return -1.0f;

  float jarak = durasi * 0.0343f / 2.0f;
  if (jarak < 2.0f || jarak > 400.0f) return -1.0f;
  return jarak;
}
03Kontrol servoGerakan perlahan antara posisi buka dan tutup 
void gerakkanServoPerlahan(int sudutTujuan) {
  sudutTujuan = constrain(sudutTujuan, 10, 170);

  if (posisiServo < sudutTujuan) {
    for (int sudut = posisiServo; sudut <= sudutTujuan; sudut++) {
      servoTutup.write(sudut);
      delay(JEDA_GERAK_SERVO_MS);
      yield();
    }
  } else {
    for (int sudut = posisiServo; sudut >= sudutTujuan; sudut--) {
      servoTutup.write(sudut);
      delay(JEDA_GERAK_SERVO_MS);
      yield();
    }
  }
  posisiServo = sudutTujuan;
}

void loop() {
  unsigned long sekarang = millis();
  prosesWiFi(sekarang);
  prosesPerintahSerial();

  if (sekarang - sensorTerakhir >= INTERVAL_SENSOR_MS) {
    sensorTerakhir = sekarang;

08 / Hasil & Pembelajaran

Dari prototype menuju sistem yang lebih matang.

Hasil Project

Prototype terintegrasi

Dua sensor, servo, audio, Wi-Fi, dan Telegram bekerja dalam satu alur sistem.

Tantangan

Stabilitas pembacaan

Dua sensor perlu diberi jeda agar tidak saling mengganggu, sedangkan status penuh perlu divalidasi berulang.

Solusi

Jeda dan hysteresis

Program memberi jeda 40 ms antarsensor, tiga pembacaan untuk perubahan status, serta batas reset 15 cm.

Hal yang Dipelajari

Integrasi perangkat

Project memperkuat pemahaman komunikasi serial, aktuator, pengukuran jarak, dan notifikasi jaringan.

Pengembangan Berikutnya

Monitoring lebih lengkap

Versi berikutnya dapat menampilkan persentase kapasitas, histori data, casing lebih kuat, dan pemilahan sampah.
